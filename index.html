Stripe Subscription Integration

View Details...

I set up a monthly subscription system using Laravel Cashier and Stripe.

Users see a dynamic section based on their current subscription status.

If they've never been subscribed, they can sign up.

If they're subscription is active, they can cancel.

If they've cancelled but their final month hasn't run out, they can resume.

They can also see their payment history and download a PDF invoice for each transaction.

If the user has previously added a card, they can update their card details.

What I learned

It's fairly straight-forward to set up a Stripe payment system with Laravel Cashier.

That's pretty much all there is to say about it.

I just followed the docs and didn't encounter any major issues i.e. learning opportunities.

Twitter-style Social Platform

View Details...

I decided to learn Vue.js by building a simple single page social platform based on Twitter.

The basic concept is that users can register, login and then create posts. They can then comment on their own or other users' posts.

The lists of posts use infinite scroll for a better user experience.

Users can click on a user's name to see more information about that user as well as a list of that users' posts.

There is also a side tab that lists the most followed users.

What I learned

Building an app in a new framework, in this case Vue, is inevitably going to cause you to learn the basics. Things like components, props, events etc.

But one of the more specific lessons I learned was to use passed-down props rather than generating what I needed within the child component itself.

For example, I wanted to find out if the current user was following the user displayed in the User component in order to determine whether to display the follow or unfollow button.

My initial idea was to run through the user's followers and add them all to a new array within each User component and then check if the user's ID is in that.

However, this didn't update the follow/unfollow button because the array was recreated before the user prop had been passed back down from the parent.

I resolved this initially by delaying the array update but this didn't feel right.

After sleeping on it, I came back and found a way to get the follower IDs straight from the user prop using the map() function.

This allowed the follow/unfollow buttons to update, even if the prop took a little while to update.

Open Street Map Integration

View Details...

For this project, I was adding a map that showed markers for certain locations provided by longitude and latitude from a database.

New locations can be created by clicking on the map.

It also has search functionality, using Nominatim, that can be used to travel to one of the saved locations or to a city, town or country.

If a user has a 'hometown' set in their profile settings, the map will load there and a bounding box can be toggled to show the area of their hometown.

What I learned

Primarily, I learned how to use the OpenLayers library to create the different layers for the map, the markers and the popups.

As well as navigating to certain points on the map.

I learned that there are many standards for latitude and longitude values and how to convert them into the standard needed.

I also learned how to integrate the Nominatim API to geocode (get lat & lon from place names) then parse & cache the results, since they aren't likely to change significantly between requests.

Automated Clean Up

View Details...

User generated content on this platform is soft deleted when the user clicks delete.

It then appears in their bin where it remains for 30 days before being automatically removed for good.

The user can either restore or permanently remove a piece of content from their bin.

Every day at 03:00, the script runs to find deleted content that was deleted more than 30 days ago and then permanently removes it.

What I learned

How to schedule commands and then set up a cron to run the schedule.
